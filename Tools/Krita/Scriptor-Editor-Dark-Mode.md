# 20260410225145 Scriptor Editor

To set the **Scripter** editor to dark mode in Krita, you need to adjust the settings within the Scripter window itself. Even if your main Krita interface is dark, the code editor sometimes defaults to a light syntax highlighting scheme.

Here is how to change it:

### 1. Open the Scripter Settings
1. Open Scripter (**Tools > Scripts > Scripter**).
2. In the Scripter window menu bar, go to **Settings**.
3. Select **Configure Scripter...**

### 2. Change the Color Scheme
1. A small dialog box will appear.
2. Look for the **Color Scheme** dropdown menu.
3. Change it from "Default" (which is often light) to **Krita Dark** or **Breeze Dark**.
4. Click **OK**.

### 3. If the background stays white (UI Theme)
If the menus and borders of the Scripter window are light, it means your entire Krita theme is set to light mode. To change the overall application theme to dark:
1. Go to the main Krita window.
2. Click **Settings** in the top menu.
3. Hover over **Theme**.
4. Select **Krita dark** or **Krita-Blender**.

### 4. Troubleshooting: Text is hard to read
Sometimes, after switching to a dark theme, the Python syntax colors (like comments or strings) might be hard to see. 
*   In the same **Configure Scripter** window mentioned in Step 2, you can manually click on the color boxes for "Comments," "Strings," or "Keywords" to pick brighter colors that stand out better against the dark background.

**Tip:** If you make a mess of the colors, there is a **Restore Defaults** button in that same configuration window to start over!
